Test Lead, IT (consultant via WhileOne AB) · Scania

Oskarshamn, Sweden · Feb 2025 – Present

Brought in through my own company, WhileOne AB, as test lead in IT for Scania's part of a large, multi-year industrial program worth several billion SEK. The landscape spans the full IT/OT stack: PLC automation (Siemens and Rockwell, over OPC-UA), SCADA (Ignition), MES (Apriso), ERP (SAP plus an in-house system) and a growing set of Scania-specific tools that keep production running. The work I'm proudest of is a virtual automation layer I built: it interfaces with the Area PLC and triggers its logic exactly as real field automation would. That lets us test the upper automation layer (the SCADA PLCs), connect the SCADA and MES systems, and effectively run the whole factory virtually — validating it before the physical factory is even built. It's virtual commissioning for a plant that doesn't exist yet. I work as a test lead with real engineering know-how, which lets me go deeper, understand the systems properly, and build the tooling needed for a better, faster, more modern way of testing. The guiding principle is test early, test often; and to test often, a test has to be cheap to run — which means automated, or at least heavily tool-supported. It also means results have to be easy to interpret and it has to be obvious what was actually tested, which is where the visualizations and state graphs I build come in. I've had zero test engineers at my disposal and limited spare capacity from delivery teams already busy with their own work, so my aim has been to offload them: give them testing tools and lightweight process rather than more overhead. Among other efforts, I coordinated the IT testing for a large firewall-migration project that arrived at very short notice, and have test-led across several projects.

Tech: Python · Test lead · Virtual commissioning · Simulation · Test automation · Siemens & Rockwell PLC · OPC-UA · Ignition SCADA · Apriso MES · SAP ERP · OT/IT integration · Integration & E2E testing · Test visualization

Software Engineer in Test · Spotify

Stockholm, Sweden · Jan 2012 – Apr 2013

At Spotify I focused on reliability and catching issues early. I hardened the business-critical data pipeline behind payments and analytics — the logic that determined who got paid, how much and when, across multiple markets and continents. It ran on what was, at the time, Europe's biggest Hadoop cluster: hundreds of compute nodes. The pipeline was a chain of large jobs where each result fed into the next, so cascading errors were a real concern — and performance was critical, because one complete run took 20 of the 24 hours available before the next day's batch of data was ready. There was no room to simply re-run a failure. To keep it reliable I built live monitoring that alerts the moment something breaks, so problems surface early instead of at hour 20, and constructed a production-like test environment where we could try new Hadoop versions, jobs and configurations before running them on the cluster. There was a fair bit of data analysis too — checking that results made sense, including comparing down-sampled output against full-data-set results. Stack: Apache Hive, Hadoop, Python, Luigi. I also started in the Web Player team and helped take it to its first public launch. (A few of us even wrote a JavaScript game that ran inside the Spotify desktop client.)

Tech: Python · Hadoop · Apache Hive · Luigi · Big data · Model Driven Testing · Monitoring · Internal tooling

Related: Custom monitoring for a 20-hour data pipeline
